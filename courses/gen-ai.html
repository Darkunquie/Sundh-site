What You'll Learn

Eight modules, ten weeks — every concept anchored by code that calls a real LLM API on day one.

  • What an LLM actually is, demystified
  • Tokenization and the cost of every prompt
  • Attention and the transformer — the intuition
  • Open vs closed models — when each makes sense
  • Model providers: OpenAI, Anthropic, Mistral, open-source
  • Zero-shot, few-shot, and chain-of-thought
  • System prompts and persona design
  • Structured output: JSON mode, schemas, validation
  • Prompt templates and reusable patterns
  • Adversarial prompting and prompt injection defence
  • Why RAG beats fine-tuning for most apps
  • Chunking strategies and metadata design
  • Embedding models and similarity search
  • Hybrid retrieval: keyword + vector
  • Evaluating RAG quality with golden sets
  • Pinecone, Chroma, Weaviate compared
  • Indexing strategies and recall vs latency
  • Metadata filtering and namespaces
  • Self-hosting Chroma for prototypes
  • Production patterns and cost control
  • LangChain core: chains, prompts, parsers
  • LlamaIndex: query engines and indexes
  • Memory, callbacks and streaming
  • LangSmith for tracing and evaluation
  • When to drop frameworks and call APIs directly
  • ReAct pattern: reason + act loops
  • Tool/function calling — OpenAI and Claude
  • Designing safe, scoped tools
  • Error handling and self-correction
  • Hands-on: a research agent with browsing
  • Single-agent vs multi-agent trade-offs
  • CrewAI and AutoGen architectures
  • Roles, planners and critics
  • Inter-agent messaging and termination
  • Where multi-agent breaks (and how to recover)
  • FastAPI backend for an LLM app
  • Streaming responses and websockets
  • Evals: regression tests for prompts
  • Cost, latency and rate-limit handling
  • Capstone: a deployed agent on your domain
